# 時間隨機地城消遣團（暫定名稱）
## 遊戲類型與核心定位

- 消遣向、無限層數地城冒險桌遊
- 輕鬆、不競技、不硬核懲罰
- 可單人刷、可2–4人連線合作
- 失敗只是「今天玩得很開心，下次再來」，可帶部分戰利品回家
- 可隨時中斷，用 seed + 狀態存檔，下次可接續或換人加入
- 重玩性來源：隨機地圖、隨機事件、詞綴刷取、每次換不同裝備組合

## 已確認的核心機制

### 1. 時間戳系統（萊莎鍊金1代風格 ATB + 共看時間軸）

- 所有單位（玩家＋怪物）有各自「速度值」（範例：8～20）
- 共用一條時間軸（從0開始一直往右累積）
- 每輪自動推進，所有單位 icon 根據速度值前進
- icon 先碰到下一個「門檻線」（基準刻度 = 100）的單位行動
- 行動完成後，該單位 icon **重置到 0**（或扣除100，待微調）
- 未來可加入加速技能/詞綴影響讀條速度或門檻值
- **同時行動處理**（非常消遣向）
  - 玩家之間同時 → 擲骰子/硬幣決定順序（高者先，平手重擲）
  - 怪物之間同時 → 隨機或擲骰
  - 玩家 vs 怪物同時 → **玩家優先**（爽感優先）

### 2. 單次行動流程（每輪輪到誰就完整執行一次）

1. **移動階段**
   - 移動格數依裝備職業特徵、被動、詞綴決定（之後細談）
   - 可中途使用「移動類道具」（如瞬移卷）

2. **行動階段**（攻擊／技能）
   - 選擇技能／普通攻擊
   - 多數技能有「讀條」時間（需等待時間戳累積）
   - 讀條完成後才可移動到目標位置施放 → 有「預測走位」樂趣
   - 可中途使用「戰鬥類道具」（如投擲炸彈）

3. **道具／互動階段**（獨立結尾階段）
   - 使用藥水、卷軸、開箱子、環境互動等
   - 使用限制分三類：
     - 房間限制（這層只能用 X 次）
     - 行動限制（本次行動內限 1 次）
     - 冒險限制（整場地城限 1～2 次）

### 3. 地圖與格子特性

- 無硬性規定長寬（可 3×3～9×9 或更大，依關卡變化）
- 格子可能具有特性（未來擴充方向）：
  - 回復格（定時產生回復球）
  - 遮蔽物（可閃避特定攻擊／AoE）
  - 上下階梯（多層結構，例如城堡廢墟、王座大廳）
- 怪物與玩家都在同一張棋盤上移動

### 4. 技能施放與預測機制（目前最有潛力的爽點）

- 大部分技能有「讀條」時間
- 讀條期間怪物會依據自身速度移動
- 玩家需預測怪物未來位置 → 讀條結束後移動到正確格子施放
- 打空懲罰輕（浪費時間戳＋可能觸發搞笑小事件）
- 速度高／連動多的角色可進行更複雜的預讀 combo

### 5. 裝備與詞綴系統（WoW護甲＋D2詞綴路線）

- 角色本體為「全能冒險者」（無固定職業）
- 每次下地城前選擇：
  - 護甲類型（布／皮／鎖／板）
  - 武器類型（單手／雙持／雙手／遠程／法器等）
  - 付魔詞綴（1～3個）
- 詞綴重點（暗黑2風格）：
  - +X 某系技能等級
  - +X 攻擊次數
  - +X 熟練度／抗性
  - 觸發額外行動
  - 冷卻縮短、擊殺回血、暴擊連動等
- 詞綴來源：打怪、打王、開寶箱
- 可做「打寶率」裝備／詞綴

### 6. 地城每層結束條件（隨機抽取）

- 殺死特定菁英／Boss
- 開啟特定機關／找到鑰匙
- 殺死所有怪物
- 存活 X 時間（時間戳累積到某值）
- 其他特殊條件（玩家投票或隨機）

### 7. 地城外準備（村莊／手遊式介面）

- 無需實體地圖，點選功能即可
- 設施可能包含：
  - 購買／製作裝備
  - 購買／合成道具
  - 學習／升級技能
  - 聊天區
  - 付魔／詞綴刷取或重roll
  - 組隊／選地圖／開房
- 房主選定地圖參數（強度係數、區域主題、額外任務等）後開啟

## 目前仍待討論／細化的項目

- 時間戳精確扣除方式（扣100 vs 歸零 vs 溢出保留）
- 移動格數具體計算公式（基礎值＋裝備＋詞綴）
- 角色基礎四屬性（力敏智耐）如何影響派生數值
- 詞綴稀有度分級與掉落機率（普通／魔法／稀有／傳奇）
- 地圖生成引擎邏輯（房間類型、連通方式）
- 多層結構（上下階梯）的具體實現方式
- 搞笑事件卡／輕鬆互動內容的數量與比重
- 單次冒險建議時長（30分鐘？1小時？還是完全自由？）

## 總結目前爽點排序（由高到低）

1. 時間戳連動＋預測走位施放的成就感
2. 每次換不同詞綴＋裝備組合的「今天玩什麼流派」期待
3. 打寶刷神詞綴的無限刷動力
4. 多人連線邊聊天邊喊「它要往左！」「打空了哈哈哈」
5. 失敗零懲罰＋隨時存檔的極致輕鬆感

3. **道具／互動階段**：使用藥水、開箱子等（獨立結尾）

- **SP**：技能點（耐力決定上限，例如每1耐=5 SP）  
- SP恢復：  
  - 每完整行動結束自動回20% SP

  - 失敗不扣，帶回家  
- 普通攻擊：0 SP  
- 大招：10–30 SP  

### 4. 地圖（MVP測試用）
- 固定5×5  
- 格子特性（未來擴充）：回復格、遮蔽物、寶箱、上下階梯  
- 怪物AI：追玩家 / 守點 / 巡邏，讀條時顯示預測路徑  

### 5. 技能施放機制
- 大多技能有基礎讀條時間  
- 讀條完成後才可移動到目標位置施放（預測走位爽點）  
- 打空懲罰輕：浪費時間戳 + 可能觸發搞笑事件

# 時間隨機地城消遣團（網頁遊戲版 - 命中/抵抗討論）

## 命中率 & 抵抗機制討論（MVP消遣定位）

**你的擔心超準！**  
- **不加命中率** → 玩家總100%命中，**太賽（太強）**，爽但無張力，預測走位變「走位就贏」。  
- **加抵抗** → 怪物有抗性/格擋，平衡好但**實作坑多**（元素抗、屬性克、隨機roll、UI顯示...網頁版一堆if-else）。  

**MVP結論：暫不加完整命中/抵抗，先用「環境 + 預測」替代平衡**（超適合消遣，爽感不減）。  
未來迭代再加簡版（1行代碼隨機miss）。

### 1. MVP方案：**零命中率，100%命中但有風險**（推薦！）
- **為什麼OK？**  
  - 核心爽點是**預測走位**（讀條猜怪物位置），不是骰子運氣。  
  - 平衡靠**環境/怪物AI**：  
    | 平衡方式 | 效果 | 網頁實作 |
    |----------|------|----------|
    | **怪物預測路徑變數** | 怪物讀條時隨機變速/轉彎（30%機率），猜錯=打空 | JS random(0.7\~1.3倍速) |
    | **格子遮蔽** | 怪物躲牆後，技能穿不過 | 射線檢測 (Phaser line of sight) |
    | **怪物閃避移動** | 速度高怪行動時自移1格 | AI狀態機 |
    | **範圍技能miss邊緣** | 3×3只傷中心，邊緣減傷50% | 距離計算 |
    | **SP/讀條懲罰** | 大招讀條長，浪費時間戳=間接miss | 已鎖定 |

- **打空視覺**：粒子「噗」 + 文字「擦身而過！」 + 小獎勵（+1金），失敗也開心。  
- **結果**：玩家感覺「我走位神」，不是「骰子爛」。

### 2. 如果要加簡版命中（迭代後，超簡單）
- **命中率公式**（1行JS）：`hitChance = 90 + (你的敏 - 怪敏)/2 %`（低於50%自動miss）。  
- **抵抗簡版**：怪物有「元素抗」（火怪抗火50%），但**只影響傷害，不是miss**（UI顯示「免疫火！」紅字）。  
  - 實作：怪物卡片標「火抗50%/冰弱20%」，傷害乘係數。  
- **隨機miss**：每攻擊roll 1d100 < 10% = miss（固定低機率，運氣元素但不主導）。

**MVP不加的好處**：  
- 代碼少（無roll UI、無抗性表）。  
- 消遣純：預測+連動>骰運氣。  
- 測試快：直接刷10層看爽不爽。

### 更新後MVP戰鬥流程（無命中）
1. 讀條 → 預測路徑（虛線，含隨機變數）。  
2. 移動施放 → **100%命中**（若位置對）。  
3. 傷害 = 基礎 × 屬性 × (1 - 格子減傷)。  
4. 怪物反擊同邏輯。

**範例**（狂戰士重擊斬）：  
- 讀條62 → 怪物預測(4,4)，**但30%機率偏移(4,3)**。  
- 你移(4,4) → 偏移miss！（粒子 + 「怪物滑步！」）  
- 下次猜準 → 力×1.5全傷，爽！

## 整合進系統
- **裝備影響**：鞋子/手套加「預測準度+10%」（減怪物偏移機率）。  
- **詞綴**：`+20%命中`（未來用，現在替「無偏移」）。  
- **職業**：敏系預測天生準（倍率影響偏移率）。

**你覺得呢？**  
- MVP**零命中**（環境平衡）OK？還是加**10%固定miss**？  
- 抵抗**只傷害減**（不block）如何？  

**下一步挑**：  
- A. **完整戰鬥範例**（無命中，秀預測偏移）。  
- B. **30個裝備清單**（WoW槽位，含被動）。  
- C. **更新MD**（全系統+此討論）。  
- D. **怪物MVP**（3種，含AI/抗性簡版）。  
- E. **村莊UI草圖**（人偶+抽證）。

## 地城關卡Config（MVP版 - 帶路線選擇機制 更新）

基於討論，我們把層與層之間的「選擇」設計成極簡、輕鬆、無博弈壓力的版本：
- 每層結束後，只給玩家**兩個路線選項**（不會超過2個，避免選擇疲勞）
- 選項描述故意籠統、模糊，讓玩家有「期待感」但無法精準預測結果
- 類似大富翁的「機會/命運」卡：系統背後隨機分配實際內容（seed決定），玩家選的只是「風格傾向」
- 沒有單局增強buff、沒有永久風險，純粹影響下一層的氛圍、獎勵/挑戰比例、怪物強度等
- 多人模式下可投票或聊天討論，增加互動

### 路線選擇機制細節
- 每層結束（清關或到達出口）時，系統生成兩個選項
- 選項名稱：固定用「艱難的路」 vs 「輕鬆的路」（或類似中性詞，避免負面）
- 描述：極短、籠統（不透露具體怪物/獎勵/風險）
  - 艱難的路：可能有更強的挑戰，但獎勵也更好
  - 輕鬆的路：相對安全，適合喘息或穩刷
- 玩家點選一個 → 系統用seed + 選擇傾向，生成下一層的實際config（theme、pool、難度等）
- 背後邏輯（不讓玩家看到）：
  - 選「艱難的路」：怪物強度+20~50%、菁英/Boss出現率↑、寶箱/掉落品質↑、環境傷害或陷阱多
  - 選「輕鬆的路」：怪物強度-10~30%、補給/回復格多、事件偏正面、但稀有掉落率↓
- 重玩性：同一seed下，選擇不同路線會產生不同體驗，玩家可分享「我選艱難結果神抽」或「輕鬆穩刷」

### 更新後的地城結構（6層範例）
- **dungeonSeed**：全域種子（決定整體風格 + 路線隨機基底）
- **floors**：陣列，共6層（每層獨立config）
- **選擇點**：在floor 1~5結束後觸發（floor 6為Boss固定結束）

範例流程（玩家視角）：
1. 清1層（戰鬥） → 出現兩個門：
   - 艱難的路（陰暗、隱約有吼聲）
   - 輕鬆的路（明亮、感覺有泉水聲）
2. 玩家選「艱難的路」 → 下一層實際變成「火山主題 + 菁英怪 + 高品質寶箱 + 熔岩傷害」
3. 清該層 → 又出現兩個選項，繼續...

### 路線選項範例（UI顯示用）
| 選項名稱       | 描述文字（玩家看到的）                     | 背後傾向（系統用，不顯示）                     | 可能實際效果範例（seed決定）                     |
|----------------|--------------------------------------------|------------------------------------------------|--------------------------------------------------|
| 艱難的路       | 前方傳來強烈的敵意，獎勵或許更豐厚...     | 難度↑、獎勵↑、風險↑                           | 火山/深淵主題、多菁英、Boss機率高、神綴掉落↑   |
| 輕鬆的路       | 看起來比較安全，似乎有恢復的氣息...       | 難度↓、補給↑、穩定性↑                         | 森林/遺跡主題、多回復格、少怪、普通寶箱多       |

- 可替換名稱（更中性/有趣）：
  - 挑戰之路 vs 安逸之路
  - 荊棘小徑 vs 平坦大道
  - 未知的誘惑 vs 安全的懷抱

### 對遊戲影響評估（結論）
- **不會影響太大**：因為結果還是系統分配，玩家無法「精算最優路線」，符合你們「消遣向、無壓力」的核心。
- **正面影響**：
  - 給玩家一點「參與感」與「期待驚喜」
  - 像大富翁抽卡一樣，有運氣成分，但不影響整體平衡
  - 多人時可以討論「要衝還是穩？」，增加樂趣
  - 重玩時可試不同選擇，體驗不同路線
- **負面風險低**：不會有「選錯就毀局」的挫折，頂多「這路線沒刷到好東西，下次換另一條」

### 實作建議（MVP）
- 每層結束時，生成兩個選項（用簡單隨機 + seed 決定哪個是「偏艱難」）
- 玩家選後，調整下一層的：
  - monsterPool 權重
  - gridTypesPool（回復/寶箱/陷阱比例）
  - environmentEffects 強度
  - difficultyScale（怪物等級、掉落品質）
- UI：兩個大按鈕 + 模糊描述 + 簡單圖示（例如火焰門 vs 綠葉門）

## 地城路線定義（MVP版 - 通則設計，可擴充地獄難度）

### 路線系統通則（核心規則，未來無限擴充）
- **生成時機**：玩家進入地城時，用 `dungeonSeed` 一次預先生成 **N 條完整路線**（MVP N=2：輕鬆+艱難；未來 N=3+ 加地獄/神級）
- **玩家互動**：預設走第1條路線（輕鬆），每層過關後給「繼續原本路線」 vs 「切換到 X 號路線」按鈕（最多切換1次，避免混亂）
- **每條路線固定6層**：層1~5可變，層6永遠Boss層
- **固定保底**：每條路線至少有1~2層補給傾向（例如層2固定補給），避免連續戰鬥崩潰
- **擴充性**：新增路線只需加新物件到路線池，系統自動用seed選取生成（例如seed決定生成「輕鬆+艱難」或「輕鬆+地獄」）
- **差異來源**：不改層類型序列，只用 **難度係數** 調整：

// 1. 簡單 hash 函數（爛但穩定，前後端共用）
function simpleHash(str) {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    const char = str.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash = hash & hash; // 32-bit
  }
  return Math.abs(hash).toString(36); // 轉成短字串 (base36)
}

// 2. 生成怪物個人 seed 的原始字串
// 用法：產生一個唯一、可重現的字串，用來當掉落隨機種子
// 輸入：floorSeed (字串), x/y (數字), index (數字，本層第幾隻怪物從0開始)
// 輸出：字串，例如 "abc123_3_2_1_4_salt2026"
function generateMonsterSeedRaw(floorSeed, x, y, index) {
  return `${floorSeed}_${x}_${y}_${index}_salt2026`; // salt 固定，防猜公式
}

// 3. 產生怪物個人 seed（最終 hash 版）
// 用法：這就是每隻怪物的「掉落種子」，用來初始化隨機
// 輸入：同上
// 輸出：短字串，例如 "k9p2m4q8r"
// 注意：同一輸入永遠一樣
function getMonsterDropSeed(floorSeed, x, y, index) {
  const raw = generateMonsterSeedRaw(floorSeed, x, y, index);
  return simpleHash(raw); // 得到穩定短字串
}

// 4. seed-based random 產生器（可重用）
// 用法：用怪物個人 seed 初始化一個 pseudo-random 函數
// 輸入：monsterDropSeed (從上面得到)
// 輸出：一個 function，每次呼叫返回 0~1 之間的隨機數，但序列固定
function createSeededRandom(monsterDropSeed) {
  let s = parseInt(monsterDropSeed, 36) || 1; // 避免0
  return function () {
    s = (s * 16807) % 2147483647;
    return s / 2147483647;
  };
}

// 5. 從掉落池抽取物品
// 用法：用 seeded random 從權重池抽 1~多個物品
// 輸入：dropPool (陣列，每項有 chance), rndFunc (從上面 create 的)
// 輸出：抽到的物品陣列
function drawDropsFromPool(dropPool, rndFunc, minCount = 1, maxCount = 3) {
  const drawn = [];
  const count = Math.floor(rndFunc() * (maxCount - minCount + 1)) + minCount; // 隨機抽幾個

  for (let i = 0; i < count; i++) {
    let total = 0;
    dropPool.forEach(item => total += (item.chance || 1));

    let r = rndFunc() * total;
    let sum = 0;
    for (let item of dropPool) {
      sum += (item.chance || 1);
      if (r <= sum) {
        drawn.push(item.item);
        break;
      }
    }
  }
  return drawn;
}

// =============================================
// 使用範例（完整流程）
function exampleMonsterDrop(floorSeed, x, y, index, dropPool) {
  // 步驟1：產生個人 seed
  const monsterSeed = getMonsterDropSeed(floorSeed, x, y, index);
  console.log("怪物個人 seed:", monsterSeed);

  // 步驟2：初始化隨機
  const rnd = createSeededRandom(monsterSeed);

  // 步驟3：抽掉落
  const drops = drawDropsFromPool(dropPool, rnd, 1, 3);
  console.log("掉落物品:", drops);

  return { seed: monsterSeed, drops };
}
