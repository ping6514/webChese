# 打寶遊戲 — 設計缺口清單（Gap Analysis）

> 基於 `newGame.consolidated.md`（MVP1 規格）。
> 以「一個可持續刷的打寶遊戲」為目標，列出**仍未定義或定義不足**的項目。
> 每項標注優先級（🔴 MVP1 必要 / 🟡 MVP2 / 🟢 後期）及目前狀態。

---

## A. 核心遊戲大循環（Meta Loop）

打寶遊戲的靈魂是「出地城 → 變強 → 再進去」的循環。目前只定義了地城內的戰鬥，缺：

| 缺口 | 說明 | 優先級 |
|------|------|--------|
| **結算畫面規格** | 層結束後：顯示什麼、帶走什麼、留什麼 | 🔴 |
| **失敗流程** | 玩家 HP=0 後發生什麼：帶走比例？回村莊？掉什麼損失？ | 🔴 |
| **村莊功能完整定義** | 目前只有名稱清單（商店/合成/抽證/倉庫），每個功能的輸入/輸出流程未定 | 🟡 |
| **地城入口條件** | 進地城前需要裝備什麼？消耗什麼？金幣/道具 | 🟡 |
| **地城難度/主題選擇** | 玩家如何選地城？解鎖條件？ | 🟡 |
| **「無限層」循環機制** | 地城是每次隨機重生成？還是固定有幾個主題地城輪流？ | 🟡 |

---

## C. 掉落池（Loot Pool）完整定義

這是打寶遊戲最核心的內容，目前只有 seed 演算法概念，完全沒有內容。

| 缺口 | 說明 | 優先級 |
|------|------|--------|
| **掉落物品種類** | 武器（基底）、綴包、金幣、消耗品、職業證碎片、裝備——各自掉落規則？ | 🔴 |
| **稀有度分級定義** | 普通 / 魔法 / 稀有 / 傳奇 的具體比例差異（數值範圍、綴數量、特殊效果） | 🔴 |
| **掉落池的來源分類** | 普通怪 / 菁英怪 / Boss / 寶箱（common/rare/legendary）各有什麼池子？ | 🔴 |
| **`dropPool` 資料結構** | `newGame.md` 只有概念範例，正式格式（包含稀有度、品質、數量範圍）未定 | 🔴 |
| **無掉落/必掉落規則** | Boss 必掉 1 件神綴？每層保底 1 個武器？無掉落層如何處理？ | 🟡 |

---

## D. 詞綴池（Affix Pool）完整定義

目前 `newGame.consolidated.md` 只有 MVP 範例（各類各 4 個），缺：

| 缺口 | 說明 | 優先級 |
|------|------|--------|
| **完整詞綴清單** | 5 類詞綴（類型/模式/特性/動作/附魔）的完整 pool，數量夠多才有重玩性（建議各類 8–15 個） | 🔴 |
| **詞綴稀有度** | 每個詞綴屬於哪個稀有度？稀有詞綴的數值範圍如何定義？ | 🔴 |
| **詞綴數值範圍** | 例如「燒傷」的 DoT 傷害是多少？固定值還是稀有度決定範圍？ | 🔴 |
| **詞綴組合限制** | 某些詞綴是否互斥？（例如「定點」+「飛越」邏輯衝突） | 🟡 |
| **詞綴合成/重隨規則** | 村莊裡「付碎片重 roll 一個綴」的具體邏輯？消耗幾個碎片？可重 roll 哪一類？ | 🟡 |

## E. 裝備系統（Equipment Slots）

文件提到「WoW 人偶 12 槽」但完全未定義槽位屬性與互動。

| 缺口 | 說明 | 優先級 |
|------|------|--------|
| **12 槽清單與各槽功能** | 頭/項/衣/手/腰/褲/鞋/戒×2/飾×1/主武/副武——每槽影響哪些數值？ | 🟡 |
| **裝備類型與生成** | 護甲/飾品/鞋子的資料結構（類似武器但不同欄位）？如何掉落？ | 🟡 |
| **裝備詞綴（非武器）** | 護甲上的詞綴（例如「+5速度」「+10%SP上限」）的池子 | 🟡 |
| **裝備品質** | 板甲/鎖甲/皮甲/布甲的差異是什麼（防禦值？職業限制？詞槽數量？） | 🟡 |

## F. 消耗品（Consumables）完整清單

目前 `effect` 欄位只是字串描述，缺少可實作的效果規格。

| 缺口 | 說明 | 優先級 |
|------|------|--------|
| **消耗品種類列表** | HP 藥水（小/中/大）、SP 充能、MP 回復、解 Debuff 卷軸、傳送卷、炸彈…… | 🔴 |
| **效果 type enum** | `effect` 需要是可解析的結構，而非純字串（例如 `{ type: "heal_hp", value: 30 }`） | 🔴 |
| **使用限制規則** | 道具的三類限制（行動/房間/冒險）的實際 counter 如何儲存與重置？ | 🟡 |
| **消耗品獲取方式** | 村莊購買價格？地城掉落率？寶箱固定給？ | 🟡 |

## L. 引擎 State 介面（給實作用）

`GameEngineDesign.md` 是象棋引擎，新遊戲的 state shape 完全未定義。

| 缺口 | 說明 | 優先級 |
|------|------|--------|
| **`GameState` TypeScript 介面** | 包含：units、grid、timeline（ATB）、floorConfig、dungeonState、globalFlags | 🔴 |
| **`Unit` 介面** | 玩家/怪物共用，包含 currentStats、weaponSlots、buffs/debuffs、atb、position | 🔴 |
| **`Action` union type** | `ADVANCE_TIME`, `MOVE`, `QUEUE_WEAPON`, `RESOLVE_CAST`, `APPLY_RECOVERY`, `USE_ITEM`, `END_TURN` | 🔴 |
| **`Event` union type** | `UNIT_MOVED`, `CAST_STARTED`, `DAMAGE_DEALT`, `BUFF_APPLIED`, `FLOOR_CLEARED`, `ITEM_DROPPED`… | 🔴 |
| **`reduce` 函式簽名** | `reduce(state, action) → { ok, state, events } | { ok: false, error }` 與現有引擎統一 | 🔴 |
